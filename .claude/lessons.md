# Lessons

- [2026-04-13] Generated a fake logo with green split coloring for brainstorm HTML → Rule: before writing any standalone HTML that includes brand elements (logo, nav, colors, typography), read `components/nav.tsx` and `lib/design-tokens.ts` first. Never invent visual styles from memory. Replicate exact values from the source files.
- [2026-04-13] Invented a new badge style for TierBadge (custom font, border, colors) when `ios-pill` already exists as the established badge pattern used on /brief, /morning, etc. → Rule: before implementing any UI element, grep the codebase for existing similar components. If one exists, use it. Never design a new visual component without checking first. The question to ask is always "does this already exist?" not "what should this look like?"
- [2026-04-22] `/api/measure-outcomes` had a 26h `created_at >= now - HORIZON_MS` filter paired with a daily cron, dropping ~97% of outcomes on the floor. Rows aged 26 to 48h between cron runs were excluded forever. Only 45 of 1807 rows resolved. → Rule: when a cron filters pending rows by age, the horizon must be at least `cron_interval + max_measurement_window + slack`. For a daily cron measuring a 24h outcome, use 72h minimum. Comments that claim "hourly granularity" must match the actual cron schedule in `vercel.json`.
- [2026-04-22] Added `cursor: "help"` to every hover-tooltip trigger for the wallet score popup. Browsers render that cursor as a question mark, which looked like an invented "?" icon and violated the established hover pattern. The existing `EvPopup` in `app/signals/feed/FeedClient.tsx` uses no cursor override on its trigger. → Rule: when adding a hover popup, copy the trigger treatment (cursor, border, affordance) from the existing popup pattern, not from general web convention. `cursor: "help"` is a visible change, not a neutral default.
