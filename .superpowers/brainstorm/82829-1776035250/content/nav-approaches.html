A — Keep as-is

5 top-level nav items. Wallets/Signals each have horizontal tabs inside the page. Simple but the tabs duplicate nav-level thinking.

Pros

  • Familiar pattern
  • No change needed

Cons

  • Tabs feel buried
  • No visual hint that sub-views exist

No in-page tabs — each sub-view is its own nav item

B — Expandable nav tree

Wallets and Signals expand in the sidebar. Each sub-view (Discovery, Top 50, Lookup, etc.) becomes its own route. No in-page tabs at all.

Pros

  • All nav in one place
  • Sub-views are first-class, bookmarkable
  • Clean main content area

Cons

  • Wider sidebar needed
  • More routes to manage

C — Nav + pill tabs

Nav stays as 5 items. In-page tabs become pill-style (more visible than underline tabs). Each section owns its sub-navigation clearly.

Pros

  • No nav changes needed
  • Tabs more scannable as pills
  • Easy to implement

Cons

  • Sub-views still not in nav
  • Tabs still feel like an afterthought
