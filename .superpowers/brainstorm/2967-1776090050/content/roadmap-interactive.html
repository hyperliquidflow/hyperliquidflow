→ Depends on Sprint 1: equity_tier column must exist in cohort_snapshots
Goal

Multi-line time-series chart showing average net long/short bias per wallet tier across the last N snapshots. Elite going bearish while Dust goes long = the divergence signal.

Bias formula
net_bias = (long_notional - short_notional)
          / (long_notional + short_notional)

Range: -1.0 (fully short) to +1.0 (fully long)
Deliverables
  • New API route /api/bias-trend — per-tier bias per snapshot, last 30-90 snapshots
  • Multi-line chart using recharts, one line per tier
  • Tier toggle checkboxes — default: Elite, Major, Large visible
  • Zero reference line at Y=0 (dashed)
  • KV cached, 5-min TTL, invalidated by refresh-cohort
  • New page /signals/bias + nav entry under Signals
Files Touched
app/api/bias-trend/route.ts (new) app/signals/bias/page.tsx (new) components/nav.tsx lib/design-tokens.ts (read only)
Chart Spec
Chart params
  • X: snapshot timestamps — "Apr 13 14:00"
  • Y: -1.0 to +1.0 — labeled "-100% Short" to "+100% Long"
  • Tooltip: all visible tier values at hover timestamp
  • Responsive width, full viewport height minus controls
API query params
  • window — number of snapshots (default 30, max 90)
  • tiers — comma-separated tier names (default: all)
  • Null tier = no wallets that snapshot (no line rendered)

  • Hover tooltip per cell

Goal

For a selected asset, show where wallet positions cluster along the price axis. Entry density, liquidation zones, and position-heavy levels at a glance. Reveals where forced selling or profit-taking is likely.

Bucket logic
60 price buckets
Range: current_price * 0.7 to 1.3
Liq estimate: avg_px * (1 - liq_buffer) for longs
Deliverables
  • New API route /api/market-radar — bucketed long/short/liq density per price level
  • Horizontal bar chart: longs extend left (green), shorts right (red)
  • Current price reference line with label
  • Liquidation zone overlay (dashed, distinct)
  • Asset selector (top 10 by OI) + tier filter
  • Stats row: total positions, long OI, short OI, avg entries
  • New page /signals/radar + nav entry

Goal

Add user accounts so that followed wallets, alert history, and paper portfolios persist across devices and browsers. Sprint 5 built everything in localStorage — Sprint 6 is the backend swap.

Auth approach

  • Wallet-connect login (sign message) — optional phase 2
  • On first login: import existing localStorage data into Supabase tables (one-time migration)
