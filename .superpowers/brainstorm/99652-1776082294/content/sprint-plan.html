SPRINT 1
Wallet Tier System
Foundation
  • Add equity_tier column to cohort_snapshots, computed from account_value
  • 7 tiers: Elite ($5M+), Major ($1M–5M), Large ($500k–1M), Mid ($250k–500k), Standard ($100k–250k), Small ($50k–100k), Micro ($10k–50k), Nano (<$10k)
  • Tier badge on wallet profile and leaderboard rows
  • Tier filter on /wallets/leaderboard
  • Tier counts on /wallets/discovery stats
  • Cron scoring job assigns tier on each snapshot write

SPRINT 5
Wallet Alerts + Paper Copy Trading
Full Product
  • Alerts: subscribe to wallet address, detect position open/close/size change between snapshots, store alert events, surface in UI
  • Paper copy trading: virtual portfolio that records what a wallet opens — at the snapshot detection price — and tracks virtual P&L in real time
  • Wallet following page: manage subscribed wallets, see their recent moves
  • Paper portfolio dashboard: open virtual positions, realized/unrealized PnL vs. actual wallet
  • Notification delivery: in-app first, Telegram optional phase 2
  • New tables: wallet_follows, alert_events, paper_positions

Cross-Session Continuity Plan

Each sprint gets: a spec doc in docs/superpowers/specs/, an implementation plan in docs/superpowers/plans/, and a memory entry tracking its status. At the start of each new session, the memory index shows which sprint is active and where we left off.

Sprint status will be tracked in docs/sprints/status.md — one line per sprint: Not Started / In Progress / Complete. The implementation plan for the active sprint is always the source of truth for what to do next.
