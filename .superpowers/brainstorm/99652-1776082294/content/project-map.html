5 Features — Project Map

These are 5 independent subsystems. They have dependencies on each other, so order matters. Here's how I'd sequence them.

1
Wallet Tier System
Foundation
Classify the top ~1200 tracked wallets into equity-based tiers. This becomes the grouping layer that powers every other feature — heatmap columns, bias trend lines, leaderboard filters.
Tier schema in Supabase
Tier labels + thresholds
Leaderboard filter UI
Wallet profile badge
No dependencies — build first.
↓
2
Bias Trend Chart
Analytics
Time-series chart showing directional bias (long/short skew) per tier over the last 7–30 days. Requires tier definitions from Step 1, and consumes existing cohort snapshot data already in Supabase.
Aggregate bias per tier per snapshot
Multi-line chart component
Lives on Signals or new Analytics page
Depends on: Step 1 (tiers) + existing cohort_snapshots table
↓
3
Global Position Heatmap
Analytics
Asset × Tier grid. Each cell = net long/short bias of that tier in that asset right now. Color intensity = conviction strength. Aggregates live positions from cohort_snapshots.
Asset × tier aggregation query
Grid visualization component
New /analytics or /signals/heatmap route
Depends on: Step 1 (tiers), live position data already tracked
↓
4
Market Radar
Complex
Price-axis visualization showing where wallet positions cluster — avg entry prices, liquidation zones, stop-loss levels — overlaid on a price chart for a selected asset. Needs position-level data aggregated by price bucket.
Price-bucket aggregation API
Horizontal bar chart on price axis
Asset selector
Liquidation zone estimation
Depends on: live position data. Tiers optional but enhances it.
Hardest visualization to build correctly. Defer until 1–3 are solid.
↓
5
Wallet Alerts + Paper Copy Trading
Full Product
The biggest lift. Two parts: (a) per-wallet alert subscriptions — notify when a tracked wallet opens/closes/changes a position; (b) paper copy trading — virtual portfolio that mirrors selected wallets' trades at detection time. This is a proper product, not a feature.
Alert subscription model
Notification delivery (browser / Telegram / email)
Virtual portfolio engine
Copy trade execution log
Performance vs. wallet comparison
Depends on: wallet tracking infrastructure. Can start planning in parallel.
Deserves its own dedicated spec + sprint. Plan separately once 1–3 are shipped.
